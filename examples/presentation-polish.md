# Example: Presentation Polish

## Input

```text
帮我把这个 PPT 优化一下，做得高级专业一点。
```

## Output Shape

规划结果：这次目标会让 agent 保留原有风格和文案含义，完成一版更专业的横版展示稿优化，并交付可打开的导出文件、逐页检查结论和剩余风险说明。

执行范围概览：

- 读取源文件和参考风格
- 建立页面级排版标准
- 修正层级、边距、重叠、字号和视觉重心
- 导出并逐页检查
- 输出成品路径和检查说明

可复制执行版（多行普通文本）

/goal 在保留原有视觉风格和内容语义不变的前提下，完成指定 PPT/横版 PDF 的一版完整可审阅优化，使每页具备清晰信息层级、稳定版面重心、足够留白、无文字重叠，并交付导出文件和逐页检查说明。

验证：读取源文件并导出优化后的横版 PDF；逐页渲染或截图检查标题、正文、图表、页码、边距、对齐、文字重叠、文字溢出和页面重心；抽查目录页是否水平和垂直居中，内容页是否在安全边距内。

预计耗时：约 2-5 小时。页数多、需要反复导出渲染检查或建立排版检查清单时会接近上限；额外时间只用于逐页视觉 QA、二轮修正和交付说明，不为耗时新增无关设计风格。

约束：不改变原有品牌风格、主题色、核心文案含义和章节顺序；不加入无关装饰；不把横版页面做成竖版报告式堆叠；不牺牲可读性追求复杂视觉。

边界：只处理用户指定的 PPT/PDF 及其派生输出文件；不修改无关文件夹；不覆盖源文件，除非用户明确要求。

非目标：不重写完整课程内容、不重新设计品牌系统、不新增营销页、不做动画和演讲者排练功能。

迭代策略：先建立页面级排版规则，再逐页修正；每轮导出后用渲染图检查重叠、边距和视觉重心，最多做 3 轮聚焦调整；最后输出成品路径、检查说明和仍需人工审美确认的页面。

完成条件：输出文件存在且可打开，关键页面经逐页检查无明显文字重叠、边距压线、目录偏位和内容拥挤问题，并给出输出路径和检查说明。

暂停条件：需要改变原始内容立场、替换版权素材、重做品牌风格、使用付费字体/素材、处理缺失源文件或用户需要人工审美取舍时暂停。
